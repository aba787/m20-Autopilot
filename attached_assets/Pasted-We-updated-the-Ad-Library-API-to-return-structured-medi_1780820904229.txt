This update helps ensure that media files for video and audio ad creatives are returned correctly, improving the completeness and transparency of Ad Library data.

New contents field (array of Content objects, max 10 items)

Refresh Token Expiration Policy Update
May 26, 2026

Amazon Ads API has updated its refresh token expiration policy to enhance security and align with industry best practices. Effective June 30, 2026, refresh tokens issued on or after that date expire 365 days from the date of advertiser consent, requiring developers to plan for re-authorization in their applications. Refresh tokens issued before June 30, 2026 are not affected and continue to function as before. They remain valid until the advertiser revokes authorization or another invalidation condition occurs.

Key Changes

Effective date: June 30, 2026
365-day expiration: Refresh tokens issued on or after June 30, 2026 expire 365 days from the date of advertiser consent
Fixed expiration: For affected tokens, the expiration date is fixed at consent time and is not extended by token usage or refresh activity
Re-authorization required: After expiration, the advertiser must grant consent again to restore access
Grandfathered tokens: Refresh tokens issued before June 30, 2026 are not subject to the 365-day expiration and remain valid as before
Action Required

Developers should update their applications to:

Track refresh token age from the date of advertiser consent for tokens issued on or after June 30, 2026
Proactively prompt advertisers to re-authorize before the 365-day expiration
Handle expired refresh token errors gracefully
Store consent date metadata alongside refresh tokens issued on or after June 30, 2026
For detailed implementation guidance, see the Refresh Tokens documentation.

Consent Update For Amazon Ads
December 1, 2025

If you transmit or make available to Amazon Ads any personal information, you must provide the country code for all data uploads globally. For data with a UK or EEA country code, you must also include a valid consent signal in addition to the country code.

Country Code

You must share a country code when passing customers' consented data to Amazon Ads. The country code is a 2-character string in the ISO 3166 format that indicates in which country the consent was given (for example, US, GB or DE).

NOTE: Data uploaded without specifying a country code will be rejected.

Consent Signal

You can either use (i) the IAB European Transparency & Consent Framework (TCF) signal(s), (ii) Global Privacy Protocol (GPP) or (iii) Amazon Consent Signal, to communicate UK and EEA user privacy choices to Amazon Ads.

NOTE: Please do not send multiple signals. If multiple are provided, Amazon Ads will use TCF and ignore ACS.

For detailed consent requirements by product, see Amazon Ads consent signals in the Amazon Ads Support Centre - Before You Begin section.
